# Raw2D v1.22.5 - React Fiber Phase

Raw2D v1.22.5 publishes the React Fiber package phase.

## Added

- `raw2d-react-fiber` as a separate package with an explicit custom reconciler boundary.
- Host config helpers for Rect, Circle, Line, Text2D, Sprite, and Group2D instances without copying PixiJS or Phaser APIs.
- Lifecycle-safe Sprite texture handling for external textures, owned textures, and `AssetGroup` lookups.
- Optional interaction bridge helpers that connect React-managed Raw2D objects to selection, camera, and controller workflows.
- Runnable React Fiber example and migration docs for moving from the older `raw2d-react` helpers.

## Notes

- React Fiber remains a thin integration layer over public Raw2D packages. Scene graph data stays in `raw2d-core`, rendering stays in renderer packages, and interaction stays optional.
- The package is intentionally low-level and transparent: consumers can inspect host instances, request renders explicitly, and keep renderer ownership outside React.
- Canvas remains the reference renderer. WebGL can be used by React consumers through the same public Raw2D object model.
- Package versions are aligned at `1.22.5` across every published Raw2D package.
- npm publish and Cloudflare deployment should be performed by the GitHub tag workflow or a maintainer with credentials.
