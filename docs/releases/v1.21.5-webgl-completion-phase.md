# Raw2D v1.21.5 - WebGL Completion Phase

Raw2D v1.21.5 publishes the WebGL completion phase.

## Added

- SVG texture rasterization helpers in `raw2d-sprite` so vector artwork becomes normal raster `Texture` data before Canvas or WebGL upload.
- Static WebGL run compaction for adjacent compatible clean shape and sprite runs, with explicit diagnostics through `compactWebGLStaticRuns`.
- Advanced `TextureAtlasPacker` diagnostics for fragmented area, outer waste, occupancy, and deterministic resize suggestions.
- Browser visual pixel coverage for WebGL Sprite, Text2D, ShapePath fallback, culling, static batches, and static cache hits.
- WebGL completion docs covering current status, remaining tradeoffs, performance reading, and release verification.

## Notes

- Canvas remains the complete reference renderer. WebGL is the batch-first performance renderer for large, texture-heavy, or mostly static scenes.
- Complex ShapePath fills still use explicit raster fallback when direct WebGL geometry is not enough.
- Text2D uses raster textures today; glyph atlas or SDF text can be added later without moving text data out of `raw2d-text`.
- Package versions are aligned at `1.21.5` across every published Raw2D package.
- npm publish and Cloudflare deployment should be performed by the GitHub tag workflow or a maintainer with credentials.
