## Checks

- Each route returns a healthy page response.
- Docs routes render visible content.
- Example routes render a canvas.
- Browser console has no runtime errors.
- Docs search can find WebGL content.
- Hinglish docs mode renders from `/doc?lang=hi`.

## Findings Fixed

- Sprite live docs could throw when async controls were inserted after the code block was removed by the focused demo panel.
- React example rendered only a canvas, which made browser content checks ambiguous.
- Browser smoke tests did not include `/examples/webgl-pipeline`.
- Example pages did not declare a favicon, which could create noisy browser resource errors.
